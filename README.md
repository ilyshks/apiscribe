# APIScribe 🚀

APIScribe — инструмент для автоматической генерации документации OpenAPI на основе анализа реального трафика к вашему веб-сервису. Он работает как прозрачный прокси-сервер, перехватывает запросы и ответы, анализирует их структуру и на основе собранных данных строит спецификацию API.

## ✨ Возможности

- **Прозрачный прокси** – перенаправляет запросы на целевой API, не требуя изменений в клиентском коде.
- **Автоматический анализ JSON-схем** – рекурсивно строит схемы запросов и ответов.
- **Группировка эндпоинтов** – объединяет пути с параметрами (например, `/users/1` и `/users/2` → `/users/{id}`).
- **Веб-интерфейс** – удобная панель с живыми логами, статистикой и таблицей эндпоинтов.
- **CLI-управление** – команды для запуска/остановки прокси, проверки статуса, генерации OpenAPI.
- **Live-логи** – WebSocket-трансляция логов в реальном времени прямо в браузер.
- **Экспорт OpenAPI** – генерация спецификации в формате JSON или YAML (через CLI или кнопку в UI).

## 🛠 Установка

### 1. Клонирование репозитория
```bash
git clone https://github.com/ilyshks/apiscribe.git
cd apiscribe
```
### 2. Создание виртуального окружения (рекомендуется)
```bash
python -m venv venv
source venv/bin/activate      # Linux/macOS
venv\Scripts\activate         # Windows
```
### 3. Установка зависимостей и самого пакета
#### Для разработки (режим editable):
```bash
pip install -e .
```
#### Для обычного использования (копирование пакета):
```bash
pip install .
```
#### После установки станут доступны команды `apiscribe` и `apiscribe-daemon`.

## 🚀 Запуск и использование

### 1. Запуск управляющего демона
#### В одном терминале выполните:
```bash
apiscribe-daemon
```
#### Демон запустится на `http://127.0.0.1:9001` и будет ожидать команд.

### 2. Запуск прокси для целевого API
#### В другом терминале (с активированным окружением) выполните:
```bash
apiscribe start http://localhost:5000
```
#### Замените URL на адрес вашего API. Прокси начнёт слушать порт 8080 (по умолчанию).

### 3. Отправка тестовых запросов
#### Теперь все запросы к `http://127.0.0.1:8080` будут перенаправляться на целевой API, а их данные – анализироваться и сохраняться.
```bash
curl http://127.0.0.1:8080/users/1
curl http://127.0.0.1:8080/users/2
curl -X POST http://127.0.0.1:8080/users -H "Content-Type: application/json" -d '{"name":"John"}'
```

### 4. Веб-интерфейс
#### Откройте браузер и перейдите по адресу:
```bash
http://127.0.0.1:9001/ui/index.html
```
#### Вы увидите панель с кнопками Start/Stop, статистикой, таблицей сгруппированных эндпоинтов и живыми логами. Нажмите Start, чтобы запустить прокси прямо из интерфейса.

### 5. Генерация OpenAPI-спецификации
#### Через CLI:
```bash
apiscribe openapi my_spec.json
```
#### Файл сохранится в папку `generated_docs`.

#### Через UI:
#### Нажмите кнопку OpenAPI в правом верхнем углу – спецификация скачается автоматически.

### 6. Остановка прокси
```bash
apiscribe stop
```
#### или кнопка Stop в веб-интерфейсе.

## 📁 Структура проекта
```text
apiscribe/
├── src/
│   └── apiscribe/
│       ├── cli/                # Интерфейс командной строки (Typer)
│       ├── core/               # Ядро: анализатор, коллектор, прокси
│       ├── daemon/             # Управляющий демон и control API
│       ├── exporters/          # Экспорт OpenAPI (JSON, YAML)
│       ├── generator/          # Генерация OpenAPI-спецификации
│       ├── models/             # Pydantic-модели
│       ├── storage/            # In-memory хранилище
│       ├── ui/                 # Веб-интерфейс (HTML+JS)
│       └── utils/              # Утилиты: кластеризация, слияние схем
├── pyproject.toml              # Конфигурация и зависимости
└── README.md
```
